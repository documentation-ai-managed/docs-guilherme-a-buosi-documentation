## Theme and Branding

Set your brand color to `#3B82F6` and add custom logos to match your project's visual identity.

<Steps>
  <Step title="Access Settings" icon="settings">
    Navigate to **Space Settings** > **Appearance**.
  </Step>
  <Step title="Update Colors" icon="palette">
    Enter your primary color as `#3B82F6`.
  </Step>
  <Step title="Upload Logo" icon="image">
    Drag and drop your logo file (PNG, SVG up to 1MB).
  </Step>
</Steps>

<CodeGroup tabs="JSON,YAML">
```json
{
  "theme": {
    "primaryColor": "#3B82F6",
    "logoUrl": "https://your-domain.com/logo.svg"
  }
}
```
```yaml
theme:
  primaryColor: "#3B82F6"
  logoUrl: "https://your-domain.com/logo.svg"
```
</CodeGroup>

<Callout kind="tip">
  Test your theme changes in preview mode before saving. This prevents disrupting live views.
</Callout>

## User Roles and Permissions

Control who can view, edit, or administer your documentation. Assign roles like Viewer, Editor, or Admin.

| Role     | View | Edit | Delete | Admin |
|----------|------|------|--------|-------|
| Viewer   | ✅   | ❌   | ❌     | ❌    |
| Editor   | ✅   | ✅   | ❌     | ❌    |
| Admin    | ✅   | ✅   | ✅     | ✅    |

<Tabs>
  <Tab title="Add User" icon="user-plus">
    1. Go to **Members** tab.
    2. Enter email and select role.
    3. Send invite link.
  </Tab>
  <Tab title="Permission Rules" icon="lock">
    Use granular rules for pages:

````javascript
const rules = {
  "/docs/private/*": ["admin"],
  "/docs/public/*": ["editor", "viewer"]
};
````

  </Tab>
</Tabs>

## Export and Import Features

Backup your content or migrate to another space using JSON exports.

<Expandable title="Advanced Export Options" default-open="false">
  Export includes pages, assets, and settings. Use filters for selective exports.
</Expandable>

<Request tabs="cURL,JavaScript" show-lines="true">
```bash
curl -X POST https://api.example.com/export \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -d '{"spaceId": "guilherme-buosi-docs", "format": "json"}'
```
```javascript
const response = await fetch('https://api.example.com/export', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' },
  body: JSON.stringify({
    spaceId: 'guilherme-buosi-docs',
    format: 'json'
  })
});
```
</Request>

<Response tabs="200">
```json
{
  "success": true,
  "downloadUrl": "https://example.com/export-guilehme-buosi-docs.json",
  "size": "2.5MB"
}
```
</Response>

For imports, upload the JSON file via the **Import** button in settings. Verify data integrity post-import.

<Callout kind="warning">
  Exports do not include version history. Use full backups for complete recovery.
</Callout>
